# Submission Checklist

Use this near the end of the challenge so nothing obvious is missed.

## Repo

- Public GitHub repository is ready
- Initial commit exists
- Final commit is pushed before submission email
- No commits are made after submission email

## Planning Artifacts

- Excalidraw or equivalent planning artifact is exported or shared
- UX flows are included
- Data model or ERD is included

## App

- MVP works end to end
- Core CRUD flow works
- Study flow works
- Empty and validation states are acceptable

## Quality

- Critical logic has basic tests
- README includes setup instructions
- README explains architecture and tradeoffs
- README notes improvements for more time

## Submission Email

- Repo link included
- Planning artifact link or PDF included
- Screen recording link included
