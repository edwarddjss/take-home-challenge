# Product Specialist Challenge

## **Objective**

Over a focused 6-hour session, you are tasked with planning, building, and polishing a small application from scratch. You will be expected to submit your planning artifacts, your source code, and a short README.

The challenge must be completed in **one continuous sitting**, not spread out over multiple days.

This ensures a consistent environment across all candidates and fairly tests your ability to **think critically and adapt quickly** under time constraints.

You are welcome to think about potential approaches beforehand, but **no planning, sketching, or pre-work** should be done prior to officially starting the challenge to maintain fairness.

**The challenge is structured into 4 phases:**

| **Phase** | **Time** | **Details** |
| --- | --- | --- |
| Planning & UX Sketching | Hour 0–1
*Give yourself a max of 1hr.* | Whiteboard your idea and UX flows using Excalidraw, FigJam, Miro, or similar. |
| Core Coding Phase | Hour 1–4
*Give yourself a max of 4hrs.* | Develop the application’s main functionality. Focus on MVP completion. |
| Finishing Touches & Testing | Hour 4–5:30 
*Give yourself a max of 1.5hrs.* | Refine UI/UX, add validation, write tests, polish code, and document |
| Submission | Hour 5:30 - 6
*Allow 30 minutes for final submission.* | Submit final code and send submission email with links. |

## **🚀 App Ideas (choose one)**

You must implement one of the following app ideas.

You are encouraged to extend or refine the core idea if you wish, but you **must deliver a fully working basic version** within the 6-hour timebox.

*Expand each heading to view a basic idea brief. The briefs are intentionally short and open-ended to allow you, as a Product Specialist, to apply your creativity and product thinking to shape the final experience.*

- **Task Tracker with Categories**
    
    Create, edit, delete tasks, each assigned a category (e.g., Work, Personal).
    
- **Simple Habit Tracker**
    
    Mark daily habits as done or missed, and view streaks for each habit.
    
- **Recipe Manager**
    
    Add recipes with title, ingredients, steps, and optionally upload an image.
    
- **Expense Tracker**
    
    Log daily expenses with amount, category, and description.
    
- **Book Wishlist**
    
    List books you want to read, mark them as “read”, and rate them.
    

- **Simple Poll Creator**
    
    Users can create a poll, share a link, and others can vote.
    
- **Mood Journal**
    
    Users log their mood daily with a rating and optional comment.
    
- **Flashcard App**
    
    Create decks of flashcards to quiz yourself on topics.
    
- **Workout Logger**
    
    Track workouts, reps, sets, and weights lifted.
    
- **Link Collection App**
    
    Save and tag favorite links (like a very basic bookmark manager).
    

## **🏁 Deliverables**

- **Planning Outputs**
    
    Share your UX diagrams/flows (PSD export or link to FigJam / Excalidraw / Miro).
    
- **Codebase**
    
    Solution developed in a framework you are comfortable with (NextJS & Tailwind preferred). 
    
    You are not required to use a database for the challenge. An in memory JSON object for data storage is sufficient.
    
    You are not required to implement user authentication for the challenge. A single user experience is sufficient.
    
    Code should be hosted on a public GitHub repo.
    
- **README file:**
    - Setup instructions included in your GitHub repo
    - Short explanation of your architecture/design choices
    - Anything you would improve with more time

## **📋 Evaluation Criteria**

| **Area** | **Weight** | **Focus** |
| --- | --- | --- |
| Planning & UX | 20% | Clear MVP, UX flows, wireframes, rough ERD. |
| Technical Execution | 40% | Working app, clean code, SOLID principles applied, git commit strategy. |
| Product Thinking | 20% | Smart feature prioritization and user experience focus. |
| Testing & Quality | 10% | Basic unit testing of critical logic. |
| Documentation & Communication | 10% | README clarity, commit messages, architectural explanation. |

## **✅ Notes**

- Focus on **timeboxing**: prioritize building a working MVP over polishing extra features.
- Focus on **good coding practices**: SOLID principles, reusable components, clean architecture, meaningful git commit messages.
- Focus on **usability**: a simple, intuitive experience is expected.
- You are playing **PM, Designer, Developer, and QA** — own it!

## **🧠 Use of AI**

- **Thoughtful use of AI is encouraged** to support your workflow.
    
    You may use AI tools to accelerate ideation, coding, or decisions — provided you are making product-led decisions and not outsourcing thinking.
    
- The goal is **smart efficiency**, not shortcuts.

To gain insight into your process and thinking as a Product Designer:

- You must **record your screen** for the full duration of the challenge (excluding short breaks if needed).
- Use any screen recording tool (e.g., Loom, OBS, QuickTime).

- Upload the recording to your preferred platform, **Google Drive** is recommended *(YouTube can cause problems because of the video length)*.

## **📋 Taking the Challenge**

1. **Select Your Idea**
    
    Choose one application idea from the provided list. 
    
2. **Setup boilerplate code**
You may setup boilerplate code prior to starting. This can include setting up your chosen framework or using a framework template, such as one of [Vercel’s templates](https://vercel.com/templates/next.js/nextjs-boilerplate). 
3. **Start the Challenge**
    
    When you’re ready to begin
    
    - Send an email to [kyle@unicornmachine.com](mailto:kyle@unicornmachine.com) stating the idea you’ve selected.
    - This email marks your **official start time**.
    - Start recording your screen.
    - Ensure you push an initial commit when starting so that your starting point is marked.
    - You must complete and submit all deliverables **within 6 hours**. The video recording can be sent as soon as possible after the 6 hour deadline because of the upload time.
4. **Develop Your Solution**
    
    Build your application following the provided brief, using good coding practices, MVP focus, and clean UX.
    
5. **Submit Your Work**
    
    When you finish, send an email to [kyle@unicornmachine.com](mailto:kyle@unicornmachine.com) with:
    
    - A **link to your GitHub repository**.
        - No commits should be made after the submission email is sent.
        - The repository will be cloned at the state of the last commit at the time of submission.
    - Your **planning and UX flows**, shared either as:
        - A PDF attachment, or
        - Publicly accessible links (ensure permissions are set so [kyle@unicornmachine.com](mailto:kyle@unicornmachine.com) can view them).
    - A **link to your screen recording**, ****Google Drive recommended.
